# GitTale

## 開発手順

### ブランチ戦略

- **デフォルトブランチ**: `main`
- **作業ブランチ**: `issues/{issue番号}`

### 開発フロー

1. **Issue の作成**
   - 新しい機能やバグ修正について Issue を作成する
   - Issue には概要のみを記載する（詳細な実装内容は記載しない）
   - 例: 「ユーザー認証機能の追加」「ログイン画面のレイアウト崩れを修正」

2. **作業ブランチの作成**
   - `main` ブランチから作業ブランチを作成する
   - ブランチ名は `issues/{issue番号}` とする
   ```bash
   git checkout main
   git pull origin main
   git checkout -b issues/123
   ```

3. **実装作業**
   - 作業ブランチで実装を行う
   - 適宜コミットを行う

4. **Pull Request の作成**
   - 実装が完了したら PR を作成する
   - PR には詳しい実装内容を記載する
     - 変更内容の詳細
     - 技術的な実装方針
     - テスト方法
     - 影響範囲
   - PR のタイトルまたは本文で対応する Issue を参照する（例: `Closes #123`）

5. **レビューとマージ**
   - レビューを受け、必要に応じて修正を行う
   - 承認後、`main` ブランチにマージする

### Issue と PR の関係

- **1つの Issue に対して 1つの PR** を作成する
- Issue: 何をするか（概要）
- PR: どのように実装したか（詳細）

| 項目 | 記載内容 |
|------|----------|
| Issue | 概要のみ（何を、なぜ） |
| PR | 詳細な実装内容（どのように） |
